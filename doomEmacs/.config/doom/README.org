* Comandos Básicos de configuración
- *~doom doctor~*
  Diagnostica problemas en la configuración y puede sugerirte cómo solucionarlo.
- *~doom sync~*
  Sincronizar las configuraciones de Doom Emacs con Emacs. Activa módulos del
  ~init.el~, descarga paquetes del ~packages.el~, y verifica que no haya
  errores.
  OJO: no carga cosas que estén en ~config.el~, para eso se necesita reiniciar.
- *~doom gc~*
  Para limpiar los paquetes y módulos que no estén cargados.

* Definir rutas y archivos

#+begin_src elisp :tangle config.el
(defvar dir/cosasEmacs   (expand-file-name "~/Emacs-things/"))
(defvar dir/docsPersonal (expand-file-name "~/Documents/Personal/"))

(defvar file/bibliografia (expand-file-name "referencias.bib"  dir/cosasEmacs))
(defvar file/diccionario  (expand-file-name "mis-palabras.pws" dir/cosasEmacs))
(defvar dir/orgRoam       (expand-file-name "Org-roam"         dir/docsPersonal))
(defvar dir/diario        (expand-file-name "Diario/"          dir/docsPersonal))

;; Crear directorios si no existen
(dolist (dir (list dir/cosasEmacs
                   dir/docsPersonal
                   dir/orgRoam
                   dir/diario))
  (make-directory dir t))

;; Crear archivos vacíos si no existen
(dolist (file (list file/bibliografia
                    file/diccionario))
  (unless (file-exists-p file)
    (write-region "" nil file)))
#+end_src

Este código se compone de dos partes. La primera es donde se definen las rutas y
los archivos. Lo genial es que define una ruta base y luego define rutas más específicas extendiendo desde las rutas base.

Y la segunda parte es para crear los directorios y los archivos si es que no
existen. Esto es muy útil para cuando se ejecuta Doom Emacs en una maquina
nueva, ya que de esta manera siempre se podrán usar todas las funciones, sobre
todo las de journal, org-roam y bibliografía.

* Aspecto visual
** Tipografía

Para Doom Emacs, hay 5 variables para controlar las fuentes. A parte de la
fuente primaria, el resto son opcionales
- ~doom-font~ es fuente primaria (monoespaciada).
- ~doom-variable-pitch-font~ es una fuente que no es monoespaciada
- ~doom-big-font~ es la fuente utilizada para =doom-big-font-mode=. Este modo
  es útil para presentaciones o video streaming.
- ~doom-symbol-font~ para /símbolos.
- ~doom-serif-font~ para la fuente ~fixed-pitch-serif~ (no sé cómo usarla).

#+begin_src elisp :tangle config.el
(setq doom-font
      (font-spec
       :family "Monaspace Xenon Frozen"
       :size 16 :weight 'regular)
      doom-variable-pitch-font
      (font-spec
       :family "Public Sans"
       :size 18)
      doom-serif-font
      (font-spec
       :family "Monaspace Argon Frozen"
       :size 16))

(custom-set-faces!
  '(italic
    :family "Monaspace Radon Frozen"
    :slant italic))
#+end_src

Una función poderosa de Emacs son los /faces/ que serían como estilos de fuente
que podemos configurar para ciertos casos. En la configuración de arriba, se
configuró una para que el texto en itálicas tenga una fuente distinta.

*** Plugin: Auto-Dark

Se trata de un plugin que permite sincronizar el modo oscuro.
Requiere instalar el paquete y configurar su comportamiento.

#+begin_src elisp :tangle packages.el
(package! auto-dark)
#+end_src

Caso específico de catppuccin

#+begin_src elisp :tangle config.el
(use-package! auto-dark
  :defer t
  :init
  ;; Usamos catppuccin en ambos slots; el flavor cambia vía hooks
  (setq! auto-dark-themes '((catppuccin) (catppuccin)))
  ;; Disable doom's theme loading mechanism (just to make sure)
  (setq! doom-theme nil)
  ;; Declare that all themes are safe to load.
  (setq! custom-safe-themes t)
  ;; Hooks para cambiar el flavor según el modo del sistema
  (setq! auto-dark-dark-mode-hook
         (list (lambda ()
                 (setq catppuccin-flavor 'mocha)
                 (catppuccin-reload)
                 (my/org-faces-mocha)))) ; <- fragmento agregado
  (setq! auto-dark-light-mode-hook
         (list (lambda ()
                 (setq catppuccin-flavor 'latte)
                 (catppuccin-reload)
                 (my/org-faces-latte)))) ; <- fragmento agregado
  (defun my-auto-dark-init-h ()
    (auto-dark-mode)
    (remove-hook 'server-after-make-frame-hook #'my-auto-dark-init-h)
    (remove-hook 'after-init-hook #'my-auto-dark-init-h))
  (let ((hook (if (daemonp)
                  'server-after-make-frame-hook
                'after-init-hook)))
    (add-hook hook #'my-auto-dark-init-h -95)))
#+end_src

Para temas que no son catppuccin

#+begin_src elisp :tangle-no
(use-package! auto-dark
 :defer t
 :init
 ;; Configure themes (Oscuro / Claro)
 (setq! auto-dark-themes '((doom-tomorrow-night) (doom-tomorrow-day)))
 ;; Disable doom's theme loading mechanism (just to make sure)
 (setq! doom-theme nil)
 ;; Declare that all themes are safe to load.
 ;; Be aware that setting this variable may have security implications if you
 ;; get tricked into loading untrusted themes (via auto-dark-mode or manually).
 ;; See the documentation of custom-safe-themes for details.
 (setq! custom-safe-themes t)
 ;; Enable auto-dark-mode at the right point in time.
 ;; This is inspired by doom-ui.el. Using server-after-make-frame-hook avoids
 ;; issues with an early start of the emacs daemon using systemd, which causes
 ;; problems with the DBus connection that auto-dark mode relies upon.
 (defun my-auto-dark-init-h ()
   (auto-dark-mode)
   (remove-hook 'server-after-make-frame-hook #'my-auto-dark-init-h)
   (remove-hook 'after-init-hook #'my-auto-dark-init-h))
 (let ((hook (if (daemonp)
                 'server-after-make-frame-hook
               'after-init-hook)))
   ;; Depth -95 puts this before doom-init-theme-h, which sounds like a good
   ;; idea, if only for performance reasons.
   (add-hook hook #'my-auto-dark-init-h -95)))
#+end_src

** Dired y dirvish

Ambos son exploradores de archivos que vienen instalados en Doom Emacs. La
peculiaridad de Dired es que es un buffer de Emacs, por lo que se pueden
realizar ediciones como si fuera un archivo de texto.

Y el caso de dirvish es que resulta más rápido para navegar y revisar archivos,
ya que tiene vista previa de archivos. Por defecto puede ver cualquier archivo
de texto, pero para imágenes y PDF hay que instalar:
 - ~vipsthumbnail~ para las imágenes.
 - ~pdftoppm~ para los PDF.


#+begin_src elisp :tangle config.el

;; Eliminar moviendo a la papelera

;; Abrir dirvish (dired pero mejor)

#+end_src

** Plugin: org-roam

En ~init.el~ ubicamos =org= y añadimos la bandera '+roam'. Ej:
#+begin_src elisp

#+end_src

Ahora hay que configurar el directorio de Roam

Hay dos cosas *MUY IMPORTANTES*:
 1. Utiliza una carpeta dedicada solamente a org-roam. Este plugin trabaja con
    metadatos que solo los archivos generados por él tienen, por lo cual
    conviene separar a las notas que no tienen dichos datos.
 2. Este plugin usa una base de datos, NO EDITES POR FUERA DE ESTE PLUGIN. de
    hacerlo, estarías desactualizando la base de datos.

También definí una carpeta anidada para los journal. En mi caso, utilizo este

** Plugin: org-roam-bibtex (orb)

Este plugin requiere de un archivo `.bib`, el cuál va a albergar la lista de
referencias que tendremos disponibles. Para tener una manera fácil de gestionar
este archivo, usaré Zotero.

Preparar el archivo de referencias:
 1. Instalar Zotero.
 2. Instalar el plugin [[https://github.com/retorquere/zotero-better-bibtex/releases/tag/v9.0.19][Better BibTex]] de Zotero.
 3. Ir a =Archivo → Exportar biblioteca=, seleccionar formato *Better BibLaTeX*,
    marcar *Keep updated* (dejar lo demás como está) y guardar el archivo en
    '~/Emacs-things/referencias.bib'.

Luego, hay que activar ~biblio~ en el apartado tools del ~init.el~

Y también hay que cargar un plugin
#+begin_src elisp :tangle packages.el
(package! org-roam-bibtex)
#+end_src

Y configurar el plugin

#+begin_src elisp :tangle config.el

;; Decirle a 'citar' dónde está la bibliografía (necesario para orb)
(setq! citar-bibliography (list file/bibliografia))

;; Decirle a orb dónde buscar las citas
(setq bibtex-completion-bibliography (list file/bibliografia))

;; Activar orb luego de que cargue org-roam
(use-package! org-roam-bibtex
  :after org-roam
  :config
  (org-roam-bibtex-mode 1))

;; indicarle el formato e interfaz de citas a orb
(setq orb-roam-ref-format 'org-cite)
(setq orb-insert-interface 'citar)

;; Crear un atajo para crear una nota de una bibliografía
(map! :leader
      (:prefix ("n r" . "+roam")
       :desc "Crear una nota de bibliografía" "b" #'orb-insert-link))
#+end_src

Un par de notas a parte:
 1. Citar puede ser útil para gestionar citas en Org y LaTeX.
 2. Además del atajo para insertar un link, está ~M-x citar-open~ para abrir un
    menú y navegar entre las citas disponibles y hacer varias acciones.
    - Abrir el enlace del repositorio del documento.
    - Abrir el PDF asociado en Emacs.
    - Crear la nota bibliográfica (igual que el atajo).

* Lenguajes
** Typst
Acorde a esta [[https://forum.typst.app/t/are-there-any-best-practices-for-how-to-do-typst-in-emacs/850][discusión del foro de Typst]], hay que instalar 'typst-ts-mode'
(tree-sitter), 'tinymist' (LSP) y activar 'eglot'. Por mi parte, también
instalaré 'typst-preview' porque me gusta la vista en tiempo real.

Hay que activar ~(lsp +eglot)~ y ~tree-sitter~ en ':tools' del ~init.el~.

Instalar los programas:
 - ~tinymist~

Añadir los respectivos paquetes:
#+begin_src elisp :tangle packages.el
(package! typst-ts-mode :recipe (:host codeberg :repo "meow_king/typst-ts-mode"))
(package! typst-preview :recipe (:host github :repo "havarddj/typst-preview.el"))
#+end_src

Instalar el lenguaje en el tree-sitter
 1. Asegurarnos de instalar los paquetes: ~doom sync~ y recargar Emacs.
 2. Dentro de Emacs ~M-x treesit-install-language-grammar~.
 3. Elegir ~typst~.
 4. Si pide URL: https://github.com/uben0/tree-sitter-typst.
 5. El resto en default.

Y configurar las opciones
#+begin_src elisp :tangle config.el

;; Asociar los archivos
(add-to-list 'auto-mode-alist '("\\.typ\\'" . typst-ts-mode))

;; Decirle a org que Typst existe
(with-eval-after-load 'org
  (add-to-list 'org-src-lang-modes '("typst" . typst-ts)))

;; Activar Elgot
(add-hook 'typst-ts-mode-hook #'eglot-ensure)

;; Configurar Elgot para typst
(with-eval-after-load 'eglot
  (with-eval-after-load 'typst-ts-mode
    (add-to-list 'eglot-server-programs
                 `((typst-ts-mode) .
                   ,(eglot-alternatives `(,typst-ts-lsp-download-path
                                          "tinymist"))))))

;; hacer que los tabs sean de 2 (igual que el formatter)
(after! typst-ts-mode
  (setq typst-ts-mode-indent-offset 2)
  (setq typst-ts-indent-offset 2))

;; Hacer funcionar typst-preview
(use-package! typst-preview
  :config
  (setq typst-preview-executable "/run/current-system/sw/bin/tinymist")
  (setq typst-preview-browser "default")
  ;; Utilzar Gnome Web
  (setq browse-url-browser-function 'browse-url-generic)
  (setq browse-url-generic-program "/run/current-system/sw/bin/epiphany")
)

;; Insertar citas de forma facil
(defun dp/citar-insert-typst-normal ()
  (interactive)
  (let ((key (car (citar-select-refs))))
    (insert (concat "#cite(<" key ">, form: \"normal\")"))))
(defun dp/citar-insert-typst-prose ()
  (interactive)
  (let ((key (car (citar-select-refs))))
    (insert (concat "#cite(<" key ">, form: \"prose\")"))))
#+end_src

Para la parte de las citas, hay que habilitar ~biblio~, ~vertico~ y ~lookup~ en
el ~init.el~ (algunas funciones ya están activadas) y configurar citar. En mi
caso ya configuré citar en [[*Plugin: org-roam-bibtex (orb)][Org-Roam-Bibtex]].

También incluí dos lineas para que me abra Web (navegador de GNOME) en lugar de
mi navegador por defecto. De esta manera, puedo separar mi navegador de internet
y dedica otra aplicación a la previsualización de documentos.
